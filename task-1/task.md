## Задание

Сделайте главный экран мобильного банка как на макете.

## Compose-компоненты

Главная Compose-функция должна только собирать экран из отдельных блоков. Не размещайте всю разметку внутри одной функции.

Создайте отдельные Compose-функции для:

- быстрого действия, например `QuickAction`;
- карточки баланса, например `BalanceCard`;
- карточки операции, например `OperationItem`.

`BalanceCard` используйте для зелёной карточки баланса в верхней части экрана. Логику показа и скрытия баланса разместите в этом компоненте.

`QuickAction` используйте для трёх пунктов раздела «Быстрые действия»: «Перевести», «Оплатить» и «Пополнить».

`OperationItem` используйте для каждой записи в разделе «Последние операции», передавая в него тип операции, название, дату и сумму.

Иконку не нужно передавать в `OperationItem` отдельно. Тип операции должен быть `enum class`, в котором хранится иконка:

```kotlin
enum class OperationType(
    @DrawableRes val iconRes: Int,
) {
    COFFEE(R.drawable.ic_coffee),
    TRANSFER(R.drawable.ic_transfer),
    SUPERMARKET(R.drawable.ic_shopping_bag),
}
```

В `OperationItem` используйте `iconRes` из переданного типа операции.

## Бизнес-логика

`BalanceCard`: при нажатии на иконку глаза баланс должен скрываться и отображаться снова при повторном нажатии.

`OperationItem`: в разделе «Последние операции» цвет суммы зависит от её значения: положительная сумма — зелёная, нулевая и отрицательная — чёрные.
